# 爆款分析技能

调用本技能时，优先使用 Redbook SOP 仓库作为主要上下文：

https://github.com/yhz1813600780-beep/Redbook

本机优先查找仓库路径：

- `C:\Users\Administrator\Documents\Codex\2026-04-30\sop-git`
- `C:\Users\Administrator\Documents\Codex\2026-04-30\https-github-com-yhz1813600780-beep-redbook\redbook-repo`

## 必读 SOP

开始分析前，先读取：

- `02-小红书SOP/08-爆款分析与借鉴/SOP-007_分析_爆款分析与借鉴流程.md`

如需生成新选题、标题、封面或发布文案，再按需读取：

- `02-小红书SOP/02-选题与内容规划/SOP-002_选题_爆款选题流程.md`
- `02-小红书SOP/03-标题与封面/SOP-003_标题_标题与封面制作规范.md`
- `02-小红书SOP/04-脚本与内容生产/SOP-006_内容_小红书营销专家工作手册.md`

## 输入处理

用户通常会提供爆款素材压缩包、截图、视频、链接、正文或素材说明。

如果提供压缩包：

1. 解压到当前任务工作区的临时目录，不要污染 SOP 仓库。
2. 读取文件名、图片、视频、文案、截图、互动数据、评论区信息。
3. 能识别多少就分析多少；缺失信息要在 Markdown 中标注“未提供”。

如果用户只提供链接或描述：

1. 先基于用户提供的信息分析。
2. 缺少标题、封面、正文、数据时，明确列为待补充项。

## 分析目标

每次必须完成以下输出：

1. 按 SOP-007 做爆款拆解。
2. 提炼可复用爆款因子。
3. 给出 10 个适合当前账号的新选题。
4. 每个新选题都包含：标题、封面文案、开头钩子、生图提示词。
5. 将完整结果整理成 Markdown，放回 SOP 仓库。
6. 如果仓库里已有同类主题，归纳到同一类目，例如毕业照、外出 vlog、旅行剪辑、AI 生图、校园纪念、情侣照、人生节点海报等。

## 归档规则

分析结果放到：

- `02-小红书SOP/08-爆款分析与借鉴/`

文件命名：

- `案例-XXX_类目_短标题.md`

如果已有同类内容，额外维护或更新：

- `分类-类目_爆款因子归纳.md`

类目归纳文件用于沉淀同类爆款的共性，必须包含：

- 同类案例列表
- 高频标题结构
- 高频封面抓手
- 高频开头钩子
- 高频视觉风格
- 可复用爆款因子
- 适合账号继续做的选题方向

新增案例编号时，先读取目录中已有 `案例-XXX_*.md`，使用下一个编号。
