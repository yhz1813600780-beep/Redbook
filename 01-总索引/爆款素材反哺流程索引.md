# 爆款素材反哺流程索引

这个索引用来说明：看到一条爆款素材后，如何在现有 Redbook 仓库里完成“收集 -> 拆解 -> 归纳 -> SOP 更新 -> 新笔记输出 -> 发布复盘”。

注意：本仓库已经有完整目录，不要另起一套 `inbox/materials/outputs` 结构。请优先使用现有的 `02-小红书SOP/08-爆款分析与借鉴/`、`05-模板库/` 和各 SOP 模块。

## 总流程

```text
爆款素材
-> 查重
-> 按模板拆解
-> 归纳爆款因子
-> 更新 SOP 或分类归纳
-> 延展自己的选题
-> 生成笔记
-> 发布复盘
-> 反哺 SOP
```

## 1. 先查重

优先查看：

- `02-小红书SOP/08-爆款分析与借鉴/README.md`
- `02-小红书SOP/08-爆款分析与借鉴/案例-*.md`
- `02-小红书SOP/08-爆款分析与借鉴/分类-*_爆款因子归纳.md`

如果已有相同或高度相似素材，不要新建重复案例。应该在已有案例或分类文件里补充新观察。

## 2. 新素材拆解

使用模板：

- `05-模板库/爆款分析模板.md`

建议保存位置：

- `02-小红书SOP/08-爆款分析与借鉴/案例-编号_主题_爆款拆解.md`

拆解时必须回答：

- 这条内容为什么值得分析？
- 标题和封面靠什么让用户点击？
- 开头靠什么让用户停留？
- 正文结构如何制造收藏、评论或转化？
- 哪些因子可以迁移到自己的账号？
- 能延展出哪些新选题？

## 3. 归纳爆款因子

如果一个规律只出现在单篇素材里，先留在案例拆解中。

如果一个规律在多篇素材中重复出现，再沉淀到：

- `02-小红书SOP/08-爆款分析与借鉴/分类-*_爆款因子归纳.md`

归纳时建议按这几类写：

- 选题因子
- 标题因子
- 封面因子
- 开头因子
- 正文结构因子
- 互动因子
- 转化因子

## 4. 反哺 SOP

当某个因子具备稳定复用价值时，再更新对应 SOP：

| 目标 | 更新位置 |
| --- | --- |
| 账号定位、人设、主页 | `02-小红书SOP/01-定位与账号搭建/` |
| 选题判断和内容规划 | `02-小红书SOP/02-选题与内容规划/` |
| 标题和封面 | `02-小红书SOP/03-标题与封面/` |
| 正文、脚本和内容生产 | `02-小红书SOP/04-脚本与内容生产/` |
| 发布检查 | `02-小红书SOP/05-发布与运营/` |
| 数据复盘 | `02-小红书SOP/06-数据复盘与优化/` |
| 爆款拆解方法 | `02-小红书SOP/08-爆款分析与借鉴/` |

## 5. 输出自己的笔记

生成新笔记时，不要照搬原素材。要保留的是：

- 选题切入
- 标题结构
- 封面逻辑
- 开头钩子
- 正文组织方式
- 评论区互动设计

需要替换的是：

- 用户场景
- 账号语气
- 案例细节
- 个人观点
- 产品或服务承接方式

## 6. 发布后复盘

复盘入口：

- `02-小红书SOP/06-数据复盘与优化/SOP-005_复盘_数据复盘与优化流程.md`
- `05-模板库/复盘模板.md`

复盘后要判断：

- 哪个爆款因子有效？
- 哪个环节失效？
- 是否需要更新分类归纳？
- 是否需要更新 SOP？
- 下一条笔记如何调整？

## 维护提醒

- 新增案例后，更新 `02-小红书SOP/08-爆款分析与借鉴/README.md` 的案例沉淀列表。
- 新增或修改 SOP 后，检查 `01-总索引/SOP总表.md` 是否需要同步。
- 结构或流程有明显变化时，更新根目录 `CHANGELOG.md`。
