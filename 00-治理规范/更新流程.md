# 更新流程

## 日常更新
1. 找到对应 SOP
2. 修改内容
3. 更新文档中的“最近更新”
4. 提交 Git

## 爆款素材分析更新
1. 收到新素材后，先检查爆款分析目录是否已有高度一致或相同案例。
2. 如果已有相似案例，先告诉用户，并给出已有 Markdown 文件链接。
3. 如果用户发送压缩包，只临时解压用于分析，不把解压文件放入仓库。
4. 分析完成、Markdown 入库并推送后，删除本次解压产生的本地临时文件。
5. 用户保留原始压缩包，仓库只维护分析文档、模板和 SOP。

## 大改动原则
- 写清楚为什么改
- 不直接删除旧方法
- 废弃 SOP 移入 `99-归档/`

## 协作建议
- 先补内容，再优化格式
- 一个提交尽量只做一类修改
- 提交信息直接写清动作和对象
