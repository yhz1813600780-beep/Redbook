# Redbook SOP 仓库

这是一个专门用于维护小红书运营 SOP 的 Git 仓库，适合自媒体团队持续更新、协作编辑和按版本回溯。

本仓库已经有既定结构，后续更新请在现有目录体系上增量维护，不要从头新建一套并行结构。

## 仓库目标
- 统一沉淀小红书运营流程
- 让选题、标题、发布、复盘有可复用标准
- 让团队成员按同一套方法执行
- 用 Git 记录每次 SOP 变化
- 把爆款素材拆解成可复用因子，反哺自己的选题、标题、封面、正文和复盘 SOP

## 仓库结构
- `00-治理规范/`：命名规范、更新规则、协作方式
- `01-总索引/`：SOP 总表和快速入口
- `02-小红书SOP/`：核心 SOP 文档
- `05-模板库/`：可复用模板
- `99-归档/`：废弃或历史版本 SOP

## 使用方式
1. 先查看 `01-总索引/SOP总表.md`
2. 按环节进入 `02-小红书SOP/`
3. 按模板补全和更新 SOP
4. 修改后提交 Git，保留历史

## 爆款素材反哺 SOP 的推荐路径

如果你是为了收集爆款素材，并把它变成自己的标准化小红书笔记，请优先按这条路径走：

1. 进入 `02-小红书SOP/08-爆款分析与借鉴/`，先检查是否已有同类案例或分类归纳。
2. 用 `05-模板库/爆款分析模板.md` 拆解新素材。
3. 把重复出现的规律沉淀到对应的“分类-xxx_爆款因子归纳.md”文件。
4. 如果这个规律可以稳定复用，再同步更新相关 SOP。
5. 用 `02-小红书SOP/02-选题与内容规划/`、`03-标题与封面/`、`04-脚本与内容生产/` 输出自己的新笔记。
6. 发布后进入 `02-小红书SOP/06-数据复盘与优化/` 做数据复盘，再反哺爆款分析与 SOP。

快速入口见：`01-总索引/爆款素材反哺流程索引.md`

## Git 建议
- `main`：稳定版本
- `update/...`：日常更新
- `experiment/...`：测试新方法

## 提交信息示例
- `feat: 新增选题流程 SOP`
- `update: 优化标题模板`
- `fix: 修正发布检查项`
- `archive: 归档旧版复盘流程`

## 给后续维护者的注意事项

- 这个仓库不是空仓库，已经有治理规范、SOP、模板、案例和分类归纳。
- 新增内容前先搜索已有案例、分类和 SOP，避免重复造一套结构。
- 如果修改了 README、索引、模板或 SOP，请在 `CHANGELOG.md` 说明改了什么、为什么改、后续使用时要注意什么。
- 对爆款素材的处理重点不是保存链接，而是提炼“可迁移的爆款因子”和“能落地到自己账号的 SOP”。
