# Redbook SOP 仓库

这是一个专门用于维护小红书运营 SOP 的 Git 仓库，适合自媒体团队持续更新、协作编辑和按版本回溯。

## 仓库目标
- 统一沉淀小红书运营流程
- 让选题、标题、发布、复盘有可复用标准
- 让团队成员按同一套方法执行
- 用 Git 记录每次 SOP 变化

## 仓库结构
- `00-治理规范/`：命名规范、更新规则、协作方式
- `01-总索引/`：SOP 总表和快速入口
- `02-小红书SOP/`：核心 SOP 文档
- `05-模板库/`：可复用模板
- `99-归档/`：废弃或历史版本 SOP

## 使用方式
1. 先查看 `01-总索引/SOP总表.md`
2. 按环节进入 `02-小红书SOP/`
3. 按模板补全和更新 SOP
4. 修改后提交 Git，保留历史

## Git 建议
- `main`：稳定版本
- `update/...`：日常更新
- `experiment/...`：测试新方法

## 提交信息示例
- `feat: 新增选题流程 SOP`
- `update: 优化标题模板`
- `fix: 修正发布检查项`
- `archive: 归档旧版复盘流程`
